# CourseForge System Architecture

## Overview
CourseForge is a modern SCORM and xAPI authoring tool that allows users to create interactive educational content and export it as compliant packages for Learning Management Systems (LMS).

## 1. Frontend Architecture
- **Framework**: React.js
- **State Management**: React Hooks (useState, useEffect, useRef)
- **Styling**: Vanilla CSS (embedded in `App.jsx` via `STYLES` string)
- **Icons**: Lucide-React
- **Key Components**:
  - `App.jsx`: Main application controller, handles slide management, block editing, and export orchestration.
  - Sidebar: Provides access to content blocks (Heading, Text, Image, Quiz, etc.) and import tools.
  - Canvas: The interactive workspace for editing slides and reordering blocks.
  - Modals: Includes the AI content generation interface.

## 2. Backend Architecture
- **Framework**: FastAPI (Python)
- **Entry Point**: `main.py` (Server configuration and CORS setup)
- **API Routes**: `app/api/routes.py`
  - `/api/upload/pptx`: Processes PowerPoint files into course blocks.
  - `/api/upload/xml`: Parses QTI XML files.
  - `/api/upload/story`: Extracts content from Articulate Storyline files.
  - `/api/ai/generate`: Integrates with Google Gemini API for automated content creation.
  - `/api/export/scorm`: Generates a fully compliant SCORM 1.2 ZIP package.
  - `/api/export/xapi`: Generates an xAPI-enabled HTML package.

## 3. Data Processing Layer
- **Processors (`app/processors/`)**:
  - `pptx_parser.py`: Extracts text and structure from PowerPoint files using `python-pptx`.
  - `story_parser.py`: Decodes Storyline project files.
  - `xml_parser.py`: Handles QTI (Question and Test Interoperability) XML.
  - `scorm_builder.py`: 
    - Converts authoring blocks into a Slide/Layer/Component runtime model.
    - Generates `imsmanifest.xml`.
    - Packages the SCORM runtime environment.

## 4. Runtime Environment
- **SCORM Runtime**: 
  - A specialized JavaScript runtime is injected into exported packages.
  - Handles LMS communication (LMSInitialize, LMSSetValue, LMSCommit, LMSFinish).
  - Manages course state, slide navigation, and mandatory interaction tracking.
- **xAPI Runtime**: 
  - Uses a lightweight script to track learner initialization and interactions.
  - Can be extended to send statements to an LRS (Learning Record Store).

## 5. External Integrations
- **Google Gemini API**: Used for AI-powered educational content generation.
- **LMS Compatibility**: Exports compatible with Moodle, Canvas, Blackboard, and other SCORM/xAPI compliant platforms.
